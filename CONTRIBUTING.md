# Contributing to AeroBid / AdLab

Thanks for your interest in contributing.

## How to contribute

1. Fork the repository
2. Create a branch for your change
3. Make your update in small, reviewable commits
4. Run relevant checks locally
5. Open a pull request with a clear summary

## Development notes

- The main backend project lives in `adlab-server`
- The admin frontend lives in `adlab-server/admin-frontend`
- Local development currently supports SQLite
- Deployment is designed around PostgreSQL + Docker Compose

## Before opening a pull request

Please try to include:

- what changed
- why it changed
- how you tested it
- screenshots if the change affects UI

## Code style

- Keep changes focused and easy to review
- Prefer small, clear commits over large mixed changes
- Follow existing project conventions where possible

## Questions

If you are unsure whether a change fits the project, opening an issue first is welcome.
